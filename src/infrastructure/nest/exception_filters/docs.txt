#
Микросервис работает в доверенном окружении, поэтому он всегда должен возвращать максимально полное и подробное описание ошибки клиентскому сервису.

#
В Nest gRPC в качестве описания ошибки можно передать только строку.
Сериализуем все нужные данные в JSON, и эту строку уже передаем.

#
Гейтвей работает во враждебном окружении.
Но на этапе отладки и он должен возвращать подробные ошибки.
На проде же уровень подробности надо понижать.

#
Гейтвей возвращает:
{
  "statusCode": 500,
  "message": "GearModel отсутствует",
  "context": "GearRepository.get",
  "payload": {
    "id": "b0503c20-3041-41b7-9ddd-13d7086213b8"
  },
  "status": 404
}
Фронт использует для анализа результата поле "status", а не "statusCode".
Это нужно для неограниченного расширения перечня наших статусов.
